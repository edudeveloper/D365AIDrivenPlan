Continuous Learning

TDD Playbook & Lessons Learned

The developer reviews the TDD and saves revisions with a versioned suffix. The agent compares originals vs. revisions, synthesises structured lessons, and creates a PR for the Tech Lead's approval.

Extensibility Decisions

"Prefer event handler over CoC when the standard method is called in multiple contexts"

Object Mapping

"For credit validation, use the CustCreditLimit framework — do not create custom"

Naming & Patterns

"In this project, use _Handler suffix for all event handler classes"

Common Mistakes

"Do not extend SalesConfirmJournalPost.postJournal — use the onPosted event instead"

Lessons are updated on GitHub via Pull Request — the Tech Lead approves the merge. Once merged, the agent absorbs the lessons into all future TDD generations. Closed feedback loop.
